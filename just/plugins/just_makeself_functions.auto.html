J.U.S.T. Makeself Functions¶

just_makeself_functions.bsh¶
makeself_defaultify¶
just makeself setup-local¶

Download and install makeself locally in Project’s CWD/build/makeself

just makeself just-project-locally¶
Arguments

  • [--tests] - Include unit tests. Calling this from another project is tricky though. For example, ./just --wrap bash -c 'JUSTFILE="${VSI_COMMON_DIR}/Justfile" just test'

Uses

  • tar - Only works with GNU Tar on the path.

Creates a just project executable using makeself. Should be called from the main project’s just project, or else directory paths will not line up. After the initial executable is added, called to makeself_add-files-locally should be made to add project files. For example:

  • Project: foo in .

  • Submodule: bar in ./external/bar

  • VSI Common in ./external/bar/external/vsi_common

  1. Call: just makeself_just-project-locally in .

  2. Get bar relative dir: local bar_rel="$(relative_path "${BAR_CWD}" .)" # NOTE: Does not start with ./

  3. Call: makeself_add-files-locally ./external/bar '"--show-transformed --transform s|^\./|./${bar_rel}/| --exclude=.git --exclude=./docs --exclude=./external --exclude=./tests"' in .

  4. Call: makeself_add-files-locally ./ '"--exclude=.git --exclude=./docs --exclude=./external/bar --exclude=./tests"' in .

All calls are made from the root repo directory (.), and the output will be in ./dist

just makeself add-files-locally¶
Arguments

  • $1 - Directory to add

  • [$2] - Extra tar flags to be passed to makeself/tar

After the initial just executable is created by calling makeself_just-project-locally, project directories are called using makeself_add-files-locally

The second argument should be used to exclude files, and in complicated situations, set up path transforms. * Simple: ${project_dir}/external/vsi_common * Complicated: ${project_dir}/external/${other_project}/external/vsi_common
